# Architecture

This project is organized around a small dependency flow. The goal is that
configuration, data models, services, and ADK agents each have one clear job.

> The diagrams below use Mermaid. They render directly on GitHub, GitLab, many Markdown previewers, and several documentation tools.

## Dependency Flow Diagram

```mermaid
flowchart TD
    A[main.py] --> B[news_agent.cli.main]
    B --> C[ConfigLoader.load config/*.yaml]
    C --> D[AppConfig]
    D --> E[run_triage query + config]
    E --> F[AgentGraphBuilder config .build]
    F --> G[CoordinatorAgent]
    G --> H[SequentialAgent]
    H --> I[ResearchAgent]
    H --> J[SummarizerAgent]
    I --> K[ResearchBundle]
    K --> J
    J --> L[TriageBrief]
    L --> M[HTML report]
```

## Dependency Flow

```text

  -> AppConfig

  -> AgentGraphBuilder(config).build()
  -> CoordinatorAgent
       -> SequentialAgent
            -> ResearchAgent
            -> SummarizerAgent
  -> TriageBrief
  -> HTML report
```

## Package Layout

```text
src/news_agent/
  agents/
    agent_builder.py       # Builds the ADK graph

    summarizer.py          # Thin ADK wrapper around SummarizationService

  models/
    config.py              # AppConfig, ModelConfig, SearchConfig, BudgetConfig
    triage.py              # ArticleRecord, ResearchBundle, TriageBrief, etc.

  services/
    config_loader.py       # Loads YAML into AppConfig

      base.py              # SearchClient protocol
```

## Runtime Relationship

```text
main.py
  Purpose: tiny project entry point for PyCharm and CLI runs.
  Owns: DEFAULT_QUERY, DEFAULT_CONFIG, DEFAULT_HTML_OUT.
  Calls: news_agent.cli.main(argv).

cli.py
  Purpose: parse user input and output JSON/report paths.
  Calls: ConfigLoader.load(), run_triage(), write_html_report().

services/config_loader.py
  Purpose: translate YAML into typed config objects.
  Reads: config/news_agent.yaml or config/news_agent_free.yaml.
  Returns: AppConfig from models/config.py.

app.py
  Purpose: create the ADK session and Runner for one query.
  Calls: AgentGraphBuilder(config).build().
  Returns: TriageBrief from session state.

agents/agent_builder.py
  Purpose: build the ADK graph and wire dependencies.
  Creates: UsageGuard, SearchClient, ResearchService, TextGenerator,
           SummarizationService, ResearchAgent, SummarizerAgent,
           CoordinatorAgent.
```

## Class Relationship

```text
AppConfig
  contains ModelConfig
  contains SearchConfig
  contains BudgetConfig
  contains list[OutletConfig]

AgentGraphBuilder
  depends on AppConfig
  builds CoordinatorAgent

CoordinatorAgent
  owns SequentialAgent

SequentialAgent
  runs ResearchAgent then SummarizerAgent

ResearchAgent
  depends on ResearchService
  writes ResearchBundle to session state

ResearchService
  depends on exactly one SearchClient

SearchClient
  implemented by OpenAIWebSearchClient
  implemented by GoogleNewsRssSearchClient

SummarizerAgent
  depends on SummarizationService
  writes TriageBrief to session state

SummarizationService
  depends on TextGenerator
  converts ResearchBundle into TriageBrief

TextGenerator
  implemented by OpenAIResponsesTextGenerator
  implemented by StaticTextGenerator
```

## Design Rule

Provider-specific code should stay at the edge:

- OpenAI search code belongs in `services/search/openai.py`.
- RSS search code belongs in `services/search/rss.py`.
- Agents should not know provider details.
- `ResearchService` should only depend on the `SearchClient` interface.
- Data-only classes belong in `models/`.
- YAML loading belongs in `services/config_loader.py`.
- `app.py` should orchestrate one run, not build provider-specific clients itself.
